10 · JWT + HttpOnly Cookie & Account Lockout

Token 不再放在 localStorage

JS 無法直接讀取 → 降低 XSS 風險;雙層授權 + 5×15 鎖定機制防暴力破解

LOGIN FLOW · 8 STEPS

JWT 登入完整流程

1前端送出 POST /auth/login
2後端驗證 email / password
3檢查帳號是否啟用 / 鎖定
4簽發 JWT · HS256 · 24h 有效
5set_cookie · httponly · samesite=lax
6axios withCredentials:true 自動帶 Cookie
7POST /auth/logout → 刪除 Cookie
8API 回 401 → 前端導向登入

密碼安全 · 認證授權 · 路由保護 · 輸入驗證 · 網路層 · 可稽核性

01 · 密碼安全

passlib[bcrypt]

  • bcrypt 密碼雜湊,不存明文
  • 強度驗證:長度 ≥ 6
  • 至少符合 3 種字元類型
02 · 認證授權

JWT + Depends

  • JWT HS256 簽章 · exp 24h
  • HttpOnly Cookie 傳輸
  • get_current_user()
  • require_admin()
03 · 前端路由保護

3 Route Guards

  • GuestOnly 阻擋已登入者
  • RequireAuth 阻擋未登入者
  • RequireAdmin 阻擋非管理員
04 · 輸入驗證

Pydantic v2

  • 強制型別驗證
  • 格式錯誤自動回 422
  • Email 格式驗證
  • Role 值域限制
05 · 網路層

CORS + 401

12 / 17SECURITY · KNOWN RISKS
12 · Security — Known Risks & Pre-Production Checklist

針對現行版做風險盤點

Demo 階段取捨;正式部署前必須處理。本頁不再涉及已淘汰架構。

高風險 JWT_SECRET 硬編碼於 config.py → os.getenv() + .env
高風險 密碼重設需確認強度檢查 → 共用 validator
中風險 sanitizeInput() 防護不足 → DOMPurify
中風險 臨時密碼亂數非密碼學安全 → crypto.getRandomValues()
低風險 密碼長度需設上限 (避免 bcrypt DoS) → max 128
低風險 後台大量查詢 API 未分頁 → limit / offset

13 / 17REACT STALE CLOSURE FIX
13 · React Stale Closure Bug Fix

debounce 搜尋永遠查不到資料

真實開發中遇到的問題 — closure 封存了當下的 state 快照

PROBLEM · 問題

症狀

  • debounce 搜尋失效
  • 搜尋永遠查不到資料
  • 輸入新關鍵字後仍以舊值送出
ROOT CAUSE
setTimeout closure 捕捉舊 state 快照
FIX · 修法

四步修正

STEP 1
新增 debouncedKeyword state
STEP 2
useEffect([keyword]) 延遲更新 debouncedKeyword
STEP 3
cleanup clearTimeout() 取消前一次計時
STEP 4
主搜尋 effect 改依賴 debouncedKeyword + 分類 + 熱量 + 頁碼
useEffect(() => { const id = setTimeout(() => { setDebouncedKeyword(keyword); }, 300); return () => clearTimeout(id); }, [keyword]);
RESULT · 驗證

Playwright 通過

✓
輸入「雞胸肉」
正確篩出對應食物
TAKEAWAY

JS closure 封存當下 state 快照 — 拆分即時輸入值與實際查詢值後,搜尋邏輯就變得穩定。

14 / 17BACKEND LAYERING
14 · Backend Layering Design

單一職責 · 七層架構

每層各有責任範圍 — Router 不直接處理底層細節,權限與資料邏輯可重用

SEVEN LAYERS

backend/ 目錄結構

main.py # FastAPI app + CORS + router 掛載 config.py # JWT / Cookie / CORS 設定 schemas.py # Pydantic v2 request/response model store.py # in-memory 資料層 singleton utils/ auth_utils.py # JWT 簽發 / 解碼 / 密碼工具 dependencies.py # Depends 權限驗證 logger.py # 操作日誌寫入 / 讀取 routers/ # auth / foods / records / profile # announcements / admin mock/ # Demo seed 資料
BENEFIT · 01

Depends 統一驗證

Router 不需重複寫 JWT 解析,登入驗證集中於 utils/dependencies.py。

BENEFIT · 02

權限邏輯可重用

get_current_user() / require_admin() 跨 API 共用。

BENEFIT · 03

資料模型集中

所有 request / response 都在 schemas.py,前後端介面清晰。

BENEFIT · 04

資料層可替換

未來換成 SQLAlchemy 時,僅須改 store.py,Router 不動。
